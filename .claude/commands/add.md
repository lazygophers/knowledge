需要处理的内容:

```
$ARGUMENTS
```

请添加、更新、完善上述的要求到 @docs/zh 中的合适位置

1. 分析需求，确认需要添加、更新、完善的文档内容
   1. 对于不清晰的部分，可以通过 `AskUserQuestion` 确认具体需求
2. 确认文档位置，判断应该添加到 @docs/zh 的哪个目录结构中
   1. 要有合理的目录结构
   2. 要有合理的层级结构
   3. 如果发现以前的结构不合理，可以通过 `AskUserQuestion` 确认是否需要调整
3. 根据需求，生成任务清单
   1. 确认需要添加、更新、完善的文档内容
   2. 确认文档位置，判断应该添加到 @docs/zh 的哪个目录结构中
   3. 需要检索的知识点
   4. 结果检查清单
   5. 确保内容都是最新的，没用过时的内容
4. 根据需求，通过 Command(deep-research) 分析，确认需要添加、更新、完善的文档内容
   1. 每一个知识点都需要根据所有的目标读者有单独的标题层级，用于针对不同的读者水平的用户区分内容、风格、复杂度等
   2. 添加丰富的图例、图表，方便读者理解和记忆
   3. 对于复杂知识点，需要有类似于海报的可视化展示，帮助读者更直观地理解
5. 更新 @docs/zh 中的文档内容
   1. 如果是新增了目录相关的部分，要及时更新相关索引

注意：

1. 所有的变更都要通过 Command(deep-research) 分析，确认无误后再提交
2. 如果有任何问题或不确定的地方，都要及时与我沟通
3. 目标读者定位
   - 无技术背景读者
   - 初级开发者
   - 中级开发者
   - 高级开发者
   - 专业开发者
